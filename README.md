# Lab – Gestion de Salle Avancée

Ce dépôt contient le projet du **Lab de Gestion de Salle Avancée** réalisé dans le cadre du cours sur MLIAEdu.  
Il s’agit d’une application permettant de gérer efficacement les salles, leurs réservations et les utilisateurs associés.

---

## 🎯 Objectifs pédagogiques
- Mettre en pratique les concepts avancés de **Hibernate/JPA**.
- Concevoir une base de données relationnelle adaptée à la gestion de salles.
- Implémenter des fonctionnalités de réservation et de gestion des disponibilités.
- Expérimenter les stratégies de cache et de préchargement pour optimiser les performances.
- Produire une documentation et des diagrammes d’architecture pour clarifier la conception.

---

## 📂 Structure du projet
- `src/main/java` : Code source des entités, services.
- `src/main/resources` : Fichiers de configuration (Hibernate, base de données, cache).
- `src/test/java` : Tests unitaires et scénarios de validation.
- `docs/` : Documentation et diagrammes (draw.io).
- `pom.xml` : Dépendances Maven.

## ⚙️ Prérequis
- **Java 17+**
- **Maven 3.8+**
- Base de données (H2 pour les tests, PostgreSQL/MySQL pour la production)
- Hibernate 6.x
- Outil de modélisation (draw.io)

## Conclusion : 

Ce laboratoire de gestion de salle avancée a permis de mettre en pratique les concepts étudiés tout au long du cours, en allant au‑delà d’une simple application CRUD.
À travers ce projet, l’étudiant a appris à :

Concevoir une base de données relationnelle adaptée à la gestion des salles et des réservations.

Implémenter des fonctionnalités avancées comme la vérification des disponibilités et la gestion des rôles utilisateurs.

Optimiser les performances grâce aux stratégies de cache et de préchargement (JOIN FETCH, EntityGraph).

Produire une documentation claire et des diagrammes d’architecture pour assurer une conception robuste et compréhensible.

Tester et valider les choix techniques afin de garantir la fiabilité du système.
